nullbot

the bot that behaves

LLM agent that runs on production servers. Every command passes through a deterministic policy engine before execution. The LLM proposes. Chainwatch enforces.

Why

LLMs hallucinate. They follow injected instructions. They escalate privileges when asked nicely. So every command nullbot wants to run passes through chainwatch — a deterministic policy engine that blocks destructive operations structurally, not probabilistically.

Nullbot investigates, diagnoses, and proposes remediation. A human approves. Then it executes — under the same enforcement.

What gets blocked

Self-protection

Nullbot cannot read its own config. Cannot dump its own environment variables. Cannot access its own API keys. Cannot modify its own binary or systemd service.

Output is scanned after every command. API keys, bearer tokens, and credential patterns are replaced with [REDACTED] before results leave the process. Binary integrity is verified at startup.

What it is not

  • Not an ML anomaly detector. Deterministic policy, no training data
  • Not a chatbot. Inbox/outbox job processing. No interactive prompts
  • Not a SaaS. Single Go binary on your server. Your data stays local
  • Not autonomous. It proposes. A human approves. Then it acts

Install

# inspect first, then run
$ curl -fsSL https://raw.githubusercontent.com/ppiankov/chainwatch/main/scripts/install-nullbot.sh -o install.sh
$ less install.sh
$ bash install.sh

# or with API key
$ GROQ_API_KEY=gsk_xxx bash install.sh

Creates system user, directories, systemd service, chainwatch policy. Runs 10-point self-protection verification. SHA-256 checksums verified on download. Read the script.

Architecture

inbox/*.json → daemon → chainwatch exec → shell → output scan → outbox/*.json ↓ ↓ LLM (redacted) policy engine ↓ allow | deny | approve

Operational constraints

1. No catastrophic blast radius. Refuse or escalate actions that can plausibly cause outage, data loss, or security breach.

2. Obey only within declared intent. Actions stay within declared scope. Out-of-scope requests are refused, not silently dropped.

3. Protect itself non-destructively. Tamper-evident. Never bricks the system. Append-only audit log with hash chain.
